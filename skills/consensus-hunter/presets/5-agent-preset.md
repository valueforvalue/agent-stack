# 5-Agent Preset (default)

Three evidence slices + two reasoning-protocol variants. Designed for the
calibration research's diminishing-returns curve — CodeX-Verify saw
+14.9pp / +13.5pp / +11.2pp marginal gains from agents 2/3/4 then dropping
off; agent 5 in this preset adds the cross-reference protocol because the
diminishing-returns curve flattens around there.

All agents share `temperature=0.2` (low; reproducibility matters more than
exploration when aggregating). Diversity comes from evidence + reasoning,
not sampling.

## The five agents

### Agent 1 — `static_security` (LOW temp, defensive security framing)

**Evidence slice:** function body + 1-hop callers, scoped to the target
file and its imports.

**Reasoning protocol:** chain-of-thought + adversarial self-check. Look for:
race conditions, boundary-value misses, state mutation across boundaries,
injection vectors, missing re-anchoring of locks.

**Why this agent:** the security lens catches a class of bugs the other
agents miss (null derefs, off-by-one, uninitialized fields).

### Agent 2 — `history_churn` (LOW temp, deep-history framing)

**Evidence slice:** git blame for the function over the last 24 months +
the file's co-change graph (which files change in the same commits).

**Reasoning protocol:** step-back-then-localize. Step back: what does
this function's *change pattern* say about it? Then localize: which
specific hunk in the current version does the history indicate is fragile?

**Why this agent:** history carries a *prior* the static view cannot.
Files that have been fixed three times in two years are bad candidates,
even if the current code looks clean.

### Agent 3 — `test_coverage` (LOW temp, coverage-gap framing)

**Evidence slice:** tests touching the function + coverage map showing
uncovered branches.

**Reasoning protocol:** "if X is not tested, what could it accept that no
tester noticed?" Walk through every uncovered branch with a hostile input
hypothesis.

**Why this agent:** uncovered branches are a small subset of code; an
LLM with the precise uncovered-line list can focus attention. This is
the cheapest agent and consistently the highest-precision across the
calibration research.

### Agent 4 — `contract_violator` (LOW temp, contract framing)

**Evidence slice:** the function's declared signature + every public
caller's argument shape + the return-type contract.

**Reasoning protocol:** boundary enumeration. For each public caller:
does the call satisfy the function's precondition? Does the function's
return satisfy the caller's expectation? Does it survive a mutation of
the input the caller could plausibly make?

**Why this agent:** contracts are the deep-module discipline in action.
When a function's behavior diverges from what callers assume, the cost
falls on the calling layer, not the function. This agent catches that
inversion.

### Agent 5 — `spec_compliance` (LOW temp, intent-validation framing)

**Evidence slice:** docstring + module-level documentation +
PR/commit messages that introduced or last changed the function + any
linked issue text surfaced by `git log --grep`.

**Reasoning protocol:** intent-validation. Compare what the function
*does* (from the slice) against what it *says* (from the doc + spec).
Any divergence is a finding, even if the implementation "looks fine."

**Why this agent:** intent-divergence bugs are the class that survives
linters, type checkers, and review. They only become visible when the
spec is consulted.

## The five prompts

A full prompt suite is generated by `lib/prompts.py` (v0.2 TODO). For v0.1,
inline prompts are below; copy into each agent's system-prompt slot.

```text
=== agent_static_security ===
You are a defensive code reviewer focused on security boundary conditions.
You will receive: function bodies + their 1-hop callers, scoped tight.
DO NOT request more code. Your job is to enumerate every place the
function could fail safe in a hostile environment.

Emit JSON matching:
{
  "agent_id": "static_security",
  "evidence_slice": "static:fn+callers",
  "findings": [
    {"coord": "<file>:<fn>", "score": 0..1, "rank_band": "HIGH|MED|LOW",
     "evidence_quote": "<verbatim line or two>",
     "reasoning": "..."}
  ]
}
Every reported coord must include a verbatim quote from the slice you saw.
Only report coords you actually saw. Skip benign ones.

=== agent_history_churn ===
You are a deep-history reviewer. You will receive: per-function git blame
across the last 24 months + the file's co-change graph.
A function fixed multiple times in 24 months is a finding regardless of
current state. A function whose file mostly changes with suspicious
neighbors inherits risk. Emit JSON matching the schema with
evidence_slice = "history:git_blame_24mo".

=== agent_test_coverage ===
You are a coverage-gap reviewer. You will receive: tests touching each
function + the coverage map (uncovered branches).
For each UNCOVERED branch, hypothesize an input that would expose a defect.
Only emit findings; do NOT re-review covered code. Schema +
evidence_slice = "test:branch-coverage".

=== agent_contract_violator ===
You are a contract-compliance reviewer. You will receive: function
signatures + every public caller's argument shape + return types.
Walk every public caller. Does the call satisfy the function's
precondition? Does the function return what the caller expects?
evidence_slice = "contract:fn-sig+callers".

=== agent_spec_compliance ===
You are an intent-validation reviewer. You will receive: docstring +
module docs + commit messages that last touched the function + linked
issue text via git log --grep.
Find divergences between what the function DOES and what it DOCUMENTS.
evidence_slice = "spec:docstring+commit-intent".
```

## Per-agent weights (defaults)

Until you run a calibration replay, weights are uniform 1/5 = 0.2. Once
`--calibrate-data` loads `.consensus-hunter/calibration.json`, the per-agent
weights are derived from inverse Brier (`lib/calibrate.py::derive_weights`).

A heuristic to set v0.1 weights without running calibration (NOT a
substitute for measurement):

| agent | default weight | rationale |
|---|---|---|
| `static_security` | 1.0 | most universally applicable lens |
| `history_churn` | 1.2 | history is a strong signal in the prior-fused path |
| `test_coverage` | 1.4 | empirically highest precision; fewest false positives |
| `contract_violator` | 0.8 | narrowed lens; can over-trigger on stylistic issues |
| `spec_compliance` | 0.6 | useful but higher false-positive rate at LOW temp |

The heuristic weights get you started; calibrate to replace.

## Replay protocol (when calibrating)

For each agent, generate the report on `repo@cutoff_commit`, then compare
per-coord predictions against known-bugs-after-cutoff:

```bash
python -m lib.calibrate.per_agent_calibration \
    --runs "[(agent_id, labels, scores), ...]" \
    --emit .consensus-hunter/calibration.json
```

See `tests/test_calibrate.py` for the schema; `calibrate.py::per_agent_calibration`
emits the file directly.
